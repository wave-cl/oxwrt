# Changelog

All notable changes to oxwrt are documented here. Follows
[Keep a Changelog](https://keepachangelog.com/en/1.1.0/) loosely:
each release is a level-2 heading dated in UTC, with commits
grouped by the `<area>:` prefix they carry (`config:`,
`rollback:`, `net:`, `firewall:`, `wan:`, etc.). Commit shas in
parentheses point back at the implementing change.

Generated by `scripts/release.sh` — hand-editing is fine but the
script will re-sort on the next run.

### Cutting a release

```sh
# Preview: groups every commit since the last tag (or all commits
# if there's no tag yet) and shows the CHANGELOG section + version
# bump without touching the repo.
scripts/release.sh patch --dry-run

# Do it. Bumps workspace version, prepends CHANGELOG entry, runs
# `cargo check --workspace`, commits, and creates an annotated tag.
scripts/release.sh patch      # 0.1.1 → 0.1.2
scripts/release.sh minor      # 0.1.1 → 0.2.0
scripts/release.sh major      # 0.1.1 → 1.0.0

# Publish:
git push origin main vX.Y.Z
```

The script prints a paste-ready release body for GitHub Releases
at the end; copy it into the web UI to match the tag.

Pre-tag history lives in `git log`. The first `release.sh` run
will land every commit since the beginning of the repo as the
inaugural CHANGELOG entry; subsequent runs only cover the range
since the previous tag.
